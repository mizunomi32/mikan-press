# スキル機能

mikan-press では、各エージェント（Research, Plan, Writer, Editor, Review）に対して独立したスキルを設定できます。スキルを使用すると、執筆スタイルやドメイン知識などをプロンプトに自動注入できます。

## 概要

- スキルは `skills/` ディレクトリに `.skill.md` ファイルとして配置
- 記事生成時に自動的に読み込まれ、該当エージェントのプロンプトに注入される
- 複数のスキルを組み合わせて使用可能
- エージェントごとに適用するスキルを指定

## SKILL.md フォーマット

```markdown
---
name: writing-style
description: 記事の文体・トーンを指定するスキル
agents:
  - writer
  - editor
---

## 文体ガイドライン

- 「です・ます」調で統一する
- 専門用語には初出時に簡単な説明を括弧書きで添える
```

### frontmatter フィールド

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `name` | string | ✓ | スキル識別子（一意の名前） |
| `description` | string | ✓ | スキルの説明 |
| `agents` | string[] | ✓ | 適用対象エージェント（`research` / `plan` / `writer` / `editor` / `review`） |

## ディレクトリ構成

```
skills/                          # プロジェクトルート
  writing-style.skill.md         # 例: 文体指定
  tech-domain.skill.md           # 例: 技術ドメイン知識
src/skills/
  loader.ts                      # スキルローダー実装
  index.ts                       # バレルファイル
```

## 使い方

1. `skills/` ディレクトリに `.skill.md` ファイルを作成
2. ファイル名は `xxx.skill.md` の形式（`xxx` は任意の名前）
3. 記事生成時に自動的に読み込まれる

### 例: 文体スキル

```markdown
---
name: casual-style
description: カジュアルな文体の記事を作成する
agents:
  - writer
  - editor
---

## 文体ガイドライン

- 「だ・である」調で統一する
- 話し言葉に近い自然な表現を使う
- 読者に語りかけるような文体で書く
- 専門用語は避け、平易な言葉で説明する
```

### 例: テックドメインスキル

```markdown
---
name: tech-domain
description: 技術記事向けのドメイン知識
agents:
  - writer
  - editor
  - plan
---

## 技術記事のポイント

- 最新の情報に基づいた内容にする
- バージョン情報を明記する
- コード例には適切なコメントを添える
- エラー対処法を具体的に記述する
- 参考リンクを適宜挿入する
```

## スキルの適用ロジック

1. SupervisorAgent が `skills/` ディレクトリから全スキルを読み込み
2. 各エージェント生成時に、`agents` フィールドに該当エージェント名が含まれるスキルを抽出
3. 抽出されたスキルをプロンプトの先頭に注入

## サンプルスキル

`skills/example-style.skill.md` にサンプルが含まれています。参考にしてください。

## 注意事項

- スキルファイルが存在しない場合、エラーにはならず機能はスキップされます
- 複数のスキルが該当する場合、ファイル名順に結合されて注入されます
- スキルの内容はプロンプトの先頭に挿入されるため、重要な指示を記述することをお勧めします
